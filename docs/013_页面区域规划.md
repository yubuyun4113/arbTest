# 监控页面 UI/UX 区域规划与设计规范

## 一、主面板结构 (Main Panel)

主面板为了信息呈现的高效与隔离，采用了现代化的 TAB 标签页视图设计：

### 1. 多 TAB 导航体系
- **TAB 1：商品套利**（涵盖黄金、原油，展现最完整的三重估值对冲链路）
- **TAB 2：纯ETF套利**（如 162411 油气、161127 生科，隐去无效的期货对应列以释放横向空间）
- **TAB 3：指数套利**（针对标普、纳指等宽基双轨拟合专属视图）
- **TAB 4：白银专区**（单独展示 161226，以应对其特殊的 VWAP 期货均价逻辑）
- **TAB 5：⚙️ 静态数据维护**（将负责底层历史数据更新的 A、B、C 控制区隔离至此，避免干扰主盯盘视线，提供配置跳转、011/012脚本手动执行及实时运行日志查看功能）

### 2. 数据表格色块视觉分离 (Color Coding)
为避免密集数据引发视觉疲劳，主面板与二级页面的列通过专门的 CSS 类进行了严谨的色块视觉分离：
*   **主面板 - 静态数据区 (`.col-static-bg`)**：赋予极浅的暖色/米黄色系背景。包含：静态官方估值、收盘价(T-1)、实时价(T-1溢价)。代表 T-1 日历史账本的固化静态数据，具有滞后性。
*   **主面板 - 实时估值区 (`.col-realtime-bg`)**：赋予浅蓝冷色背景。包含：ETF实时估值、期货校准实时估值、纯期货映射实时估值。代表融入了当前瞬间盘口变动的动态数据，指导当下套利。
*   **历史对账明细页**：对 ETF 相关列赋予**极浅薄荷绿**背景，对期货相关列赋予**极浅绯红**背景，一目了然消除盯盘视觉疲劳。

### 3. 布局、热区与字体美化规范
- **消除实时点击的混淆热区**：原先的三个分离 `<td>`（代表三套估值）独立 Hover 会在视觉上造成撕裂。系统采用了 `colspan` 结合 Flex 容器的技术（如 `combined_realtime_td_main`），将三个小单元格合并为一个整体的大 `<td>` 热区，鼠标移入整体背景加深，点击即可无歧义地进入同一套预测沙盘。
- **表格铺满底边**：将 `max-height` 高度计算逻辑放宽到 `calc(100vh - 220px)`，最大化利用屏幕垂直可视区域。
- **标红大标题**：系统主标题颜色切换为标准的 A股涨色 `#d32f2f` 并适当放大字号。
- **顶栏字体升级**：将原先硬编码的字号剥离，系统文字统一应用现代化的无衬线字体 `var(--font-sans)`，数值强制应用等宽字体 `var(--font-mono)`，确保数字高频跳动时页面不抖动。
- **现代视效**：摒弃生硬的传统表格边框，大规模引入大圆角 (`border-radius`)、柔和投影 (`box-shadow`)。

### 4. LOF 行情数据源与下单通道设计
主面板与沙盘页面分别承担了**数据获取**与**实盘交易**的职责，各自配备独立的多通道选择器：

| 层级 | 位置 | 选择器 | 用途 |
|------|------|--------|------|
| **主面板** | 顶部标题栏 | `[通达信新版 / 银河QMT / 新浪API]` | 获取 **LOF 基金实时行情**数据，数据源断开时可手动切换或点击"🔄 重连"按钮 |
| **沙盘页面** | 下单区 | `[银河QMT / 通达信]` | 执行 **A股实盘下单**，银河QMT直连`127.0.0.1:8888`，通达信使用`tqcenter.py`接口 |

> **说明**：主面板的行情数据源与沙盘的下单通道**相互独立**，任选其一即可正常工作，无需一一对应。

## 二、二级沙盘页面 (Sandbox) 与极致沉浸化

当用户在主面板点击任何一列“实时估值”单元格时，均会合并跳转至该基金唯一的“实时估值计算器（Sandbox）”二级页面。

**极致沉浸化视觉设计**：系统将最顶部的“页头控制板”与“IB夜盘数据”整块 HTML 代码挪动到了 `id="page-home"` 内部。这意味着，一旦用户点击打开沙盘（隐藏 Home 展现 Secondary），整个监控主界面顶部的冗余元素都会全部褪去，只留下极其干净专注的沙盘界面铺满全屏！

### 1. 响应式 Flexbox 布局
沙盘全面重构为 CSS Flexbox 弹性盒模型，保证在不同尺寸屏幕下，各业务区块自动居中、智能等距分布或自动换行。

### 2. 沙盘功能区域流转
沙盘页面从上至下承载了严密的套利推演逻辑：
1.  **基准数据区**：精简了提示文案，铺底展示 T-1 日的基础设施参数（基准净值、基准汇率、各 ETF 收盘价）。
2.  **LOF 价格区**：提供 A 股标的测试单价输入框，默认吸取主面版当前实盘价。该输入值将同时驱动下方所有模型的溢价测算。
3.  **三列估值推演区**：采用三种不同底色（蓝、橙、绿）并排呈现三种核心算法的当前估值与预期溢价率，并在面板上方紧凑植入对应的实盘价测试框方便推演：
    *   **① ETF实时估值 (蓝)**：
        *   **算法逻辑**：针对商品等多底层资产基金采用**加权平均矩阵算法**。获取配置的底层美股 ETF（如 USO, USO-EU 等）实时价格，分别除以其 T-1 日基准价，并按底层权重加权求和，精准还原净值波动。
        *   **底层公式**：`实时估值 = 基准净值 × [1 + 仓位 × (Σ(实时ETF/基准ETF × 权重) × (实时汇率/基准汇率) - 1)]`
        *   *(注：纯ETF及指数若存在 Woody 魔法因子，则触发 O(1) 极速计算捷径；商品多资产强制走此矩阵兜底)*
    *   **② 期货校准实时估值 (橙)**：
        *   **算法逻辑**：借助 Woody 的 `Calibration` (校准值)，先将全天候交易的期货价格“降维”映射为等效的现货 ETF 价格，随后**代入前述的加权矩阵中分配算总涨幅**，彻底避免单一主锚点导致权重失真。
        *   **底层公式**：先算 `等效ETF价 = 实时期货价 / 校准值`，进而代入：`基准净值 × [1 + 仓位 × (Σ(等效ETF价/基准ETF × 权重) × (实时汇率/基准汇率) - 1)]`
    *   **③ 纯期货映射实时估值 (绿)**：
        *   **算法逻辑**：完全抛弃现货组合与复杂的权重数组，直接拿单一主连期货合约进行**“一比一平替”**。
        *   **底层公式**：`实时估值 = 基准净值 × [1 + 仓位 × ((实时期货价 / T-1日期货结算价) × (实时汇率/基准汇率) - 1)]`
        *   *(注：该模型简单暴力，完美避开现货组合中部分衍生资产流动性枯竭的问题，专供低滑点的纯期货对冲打单)*
4.  **对冲数量与盘口区**：
    *   内置核心推算引擎：输入投入本金后，自动推导出精确到股/手的 LOF 与美股对冲标的下单数量。
    *   紧跟对冲标的（如 GLD, CL）在 IB 外盘的最新买卖一价，且支持点击价格标签“一键填入”下方限价表单。
5.  **实盘交易下单区**：
    *   外盘端直连 IB Gateway 发送委托。
    *   **A 股三通道设计**：
        *   银河 QMT：直连本地 `127.0.0.1:8888`，支持完整下单功能。
        *   通达信：使用 `tqcenter.py` 新版接口，支持完整下单功能。
        *   ~~国金 QMT：~~ **已禁用**，前端下拉菜单已屏蔽，提示"(国金QMT暂不支持)"，后端初始化与下单逻辑均已注释。
    *   **通道切换**：输入区前置 `[银河QMT/通达信]` 下拉选择器，实现银河 QMT 与通达信的灾备双通道热切换。
    *   **按钮拉开间距与原生安全弹窗**：视觉上拉开各方向下单按钮的间距防误触。在拦截层利用系统 `confirm()` 配合彩色 Emoji (如 `🔴【通达信】`) 提供最终确认，防范实盘资金误操作。

### 3. 对账明细表 (History)
当点击主面板的“静态官方估值”列时，进入的则是纯粹用于验算的对账明细二级页。如前述色块分离规范，该表同样沿用了 ETF 链路 (极浅薄荷绿) 与 期货链路 (极浅绯红) 的列底色区隔设计。
