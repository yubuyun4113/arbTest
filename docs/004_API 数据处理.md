# API 数据处理指南

## API 概述

LOF 基金套利监控系统通过多种API和网页爬虫 获取数据，包括汇率数据、LOF 基金数据、指数数据和期货数据。系统采用了多源数据获取策略，确保数据的可靠性和准确性。

## 数据来源

### 0. Woody API

字段含义

| API字段         | 描述      | 处理方式             |
| ------------- | ------- | ---------------- |
| date          | 基准日日期 | **核心时间锚点**，直接决定数据写入 basic 表的哪一行 |
| netvalue      | 基金净值    | 用于填充SPY和QQQ的净值字段 |
| CNYholdings   | 基准日汇率   | 保留作为参考，**不再作为日期判断依据** |
| CNY           | 估值日汇率   | 用于参考，不作为主要日期判断依据 |
| symbol\_hedge | 对冲ETF信息 | 包含ETF价格和权重信息     |
| calibration   | LOF校准值  | 用于填充对应LOF基金的校准字段 |
| hedge         | 对冲值     | 用于计算LOF和ETF的数量对应 |

### 1. 汇率数据

- **国家外汇管理局**：获取人民币中间价，每个交易日九点一刻公布，当天不再变动
- **新浪财经**：获取在岸价实时汇率，时刻在交易、变动
- **Woody 网页**：作为备用数据源

### 2. LOF 基金数据

- **新浪财经**：获取基金收盘价和历史数据
- **东财**：获取基金净值

### 3. 指数数据

- **新浪财经**：获取标普500和纳斯达克100指数的昨日收盘价

### 4. 期货数据

- **新浪财经**：获取黄金GC、原油CL、标普ES、纳指NQ的上一个交易日收盘价settle price
- **CME API**：采用网页爬虫获取期货的历史五日结算价


### 5. 美股数据

- **IB Gateway**：获取实时美股夜盘的ETF交易价格
- **新浪财经**：获取美股ETF的昨日收盘价

### 6. 港股数据

- **腾讯港股API**：获取港股股票（00700、00883、09988等）的历史行情数据

**API格式**：
```
https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?_var=kline_dayqfq&param=hk{symbol},day,,,320,qfq&r=0.1
```

**返回格式**：
```json
kline_dayqfq={"code":0,"msg":"","data":{"hk00700":{"day":[["日期","开","高","低","收","量"], ...]}}}
```

**字段说明**：
| 字段 | 说明 |
|-----|------|
| date | 日期 |
| open | 开盘价 |
| high | 最高价 |
| low | 最低价 |
| close | 收盘价 |
| volume | 成交量 |

**代码示例**：
```python
def fetch_hk_stock_history(symbol, start_date, end_date):
    url = f"https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?_var=kline_dayqfq&param=hk{symbol},day,,,320,qfq&r=0.1"
    response = requests.get(url, timeout=15, verify=False, proxies={"http": None, "https": None})
    text = response.text.strip()
    if text.startswith('kline_dayqfq='):
        text = text[len('kline_dayqfq='):]
    data = json.loads(text)
    day_data = data['data'][f'hk{symbol}']['day']
    df = pd.DataFrame(day_data)
    df.columns = ['date', 'open', 'high', 'low', 'close', 'volume']
    return df
```

## 数据存储

- **数据库存储**：将处理后的数据存储到 SQLite 数据库
- **文件存储**：将woodyAPI 数据存储为 CSV 和 JSON 文件，将API数据保存到`data/woodyAPI`目录，文件名格式为`Data_woodyAPI_YYYYMMDD_HHMM.json`和`Data_woodyAPI_YYYYMMDD_HHMM.csv`作为备份
- **缓存存储**：对频繁访问的数据进行缓存

## 数据更新策略

### 1. 定时更新（尚未实现）

- **汇率数据**：每日更新
- **LOF 基金净值**：每日更新（通常在交易日结束后）
- **LOF 基金价格**：每日更新（交易日结束后）
- **指数数据**：每日更新
- **期货数据**：每日更新（结算后）

### 2. 实时更新

- **LOF 基金价格**：交易日实时（银河QMT或通达信或者国金QMT）更新
- **美股数据**：交易日夜盘 IB gateway API接口 实时更新
- **期货数据**：交易日新浪接口 实时（有15秒左右的间隔）更新


## API 访问管理

### 1. 访问频率控制

- **API 限流**：根据不同 API 的限制设置访问频率
- **请求间隔**：在请求之间设置适当的间隔，避免触发 API 限制
- **批量请求**：使用批量请求减少 API 调用次数

### 2. 错误处理

- **重试机制**：当 API 请求失败时，自动重试
- **故障转移**：当主数据源失败时，切换到备用数据源
- **错误日志**：记录 API 错误，便于排查问题

### 3. 认证管理

- **woody API 密钥管理**：安全存储 woody API 密钥

## 故障排除

### 1. API 访问失败

- **检查网络连接**：确保网络连接正常
- **检查 API 状态**：确认 API 服务是否正常
- **检查 API 密钥**：确认 API 密钥是否有效
- **检查访问频率**：确认是否超过 API 访问限制

### 2. 数据质量问题

- **检查数据源**：确认数据源是否可靠
- **检查数据处理**：确认数据处理逻辑是否正确
- **检查数据存储**：确认数据存储是否正常

### 3. 性能问题

- **优化 API 访问**：减少 API 调用次数，使用批量请求
- **优化数据处理**：使用更高效的数据处理方法
- **优化存储**：使用适当的存储策略

通过合理的 API 数据处理策略，系统能够获取高质量的数据，为套利策略提供可靠的数据支持。
