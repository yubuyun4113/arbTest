## 概述

本系统支持两种QMT通信方式，用户可根据自身券商选择合适的方案：

1. **银河QMT Socket通信**：采用Client-Server架构，服务端运行在QMT策略中，客户端通过Socket通信
2. **国金MiniQMT xtquant通信**：直接使用xtquant API模块，无需Socket中转

---

## 方案对比

| 特性 | 银河QMT Socket | 国金MiniQMT xtquant | 通达信内存注入 |
|------|----------------|---------------------|----------------|
| **架构** | Client-Server | 直接API调用 | 内存注入 |
| **服务端位置** | QMT策略内 | 无需服务端 | 无需服务端 |
| **通信方式** | Socket (127.0.0.1:8888) | 直接函数调用 | 直接函数调用 |
| **延迟** | 稍高（Socket中转） | 低（原生API） | 中 |
| **复杂度** | 较高（需要维护服务端） | 较低（直接调用） | 中 |
| **适用券商** | 银河证券QMT | 国金证券MiniQMT | 支持通达信的券商 |
| **并发控制** | 互斥锁保护 | API内置控制 | 需自行控制 |
| **依赖** | threading, socket | xtquant模块 | tqcenter API |

## 🏦 交易通道配置与对接指南
系统目前支持三种 A股/LOF 交易通道，和一种美股交易通道。在前端网页的“交易沙盘”中，您可通过下拉框随意切换实盘通道。

1. 🟡 国金证券 MiniQMT (xtquant 原生直连)
- **原理**：直接调用 QMT 官方提供的 Python 极速底层库 `xtquant`。无需在 QMT 客户端中运行任何策略，系统直接从外部向 C++ 核心发单。
- **配置要求**：
  - **安装路径**：默认配置为 `你的安装目录`（需包含 `bin.x64` 和 `userdata_mini` 目录）。
  - **资金账号**：
  - **运行前提**：启动程序前，**必须先登录并保持国金 MiniQMT 客户端开启**。
- **优势**：极速、稳定、支持同步获取订单编号（seq）和详尽的资金回调。

2. 🔵 银河证券 QMT (Socket 8888 桥接)
- **原理**：部分券商（如银河）客户端对外部 `xtquant` 库进行了封锁，只能在 QMT 客户端内部编辑器中运行 Python 代码。我们采用在客户端内部建立一个 TCP Socket 服务器（端口 8888）来接收外部指令。
- **配置要求**：
  - **客户端设置**：需要在银河 QMT 策略编辑器中加载并运行 `readers/qmt_socket_server.py`。
  - **通信协议**：主程序通过 `127.0.0.1:8888` 发送诸如 `BUY,162411.SZ,100,1.520` 的文本指令。
- **优势**：能突破券商的安全封锁，泛用性极强。

3. 🔴 通达信 (极速内存注入)
- **原理**：依托于通达信的 `tqcenter` 插件 API，在同进程内存中直接操作通达信客户端的挂单窗口。
- **配置要求**：
  - **安装路径**：`安装目录\PYPlugins\user`。
  - **运行前提**：通达信客户端需登录并常驻后台。
- **优势**：适合作为 QMT 的后备容灾方案。并且兼具拉取 A 股实时快照的行情兜底功能。

4. 🌍 盈透证券 IB (Gateway / TWS API)
- **原理**：连接本地盈透交易网关，用于处理美股 ETF 现货（如 GLD, USO, SPY）和相关 CME 期货的对冲端下单。
- **配置要求**：
  - **端口**：系统会自动尝试遍历 `4001`, `4002`, `7496`, `7497`，无缝兼容 Gateway 与 TWS。
  - **设置**：客户端需要在“API设置”中勾选“启用 ActiveX 和 Socket 客户端”。
  - **盘前盘后**：系统发单时已默认附带 `outsideRth = True`，支持触发夏令时 08:00 开始的 OVERNIGHT 夜盘时段成交。

**指令协议**：

| 指令 | 格式 | 说明 |
|------|------|------|
| PING | `PING\n` | 心跳检测 |
| QUERY_TICK | `QUERY_TICK,code\n` | 查询tick数据 |
| BUY | `BUY,code,volume,price\n` | 买入下单 |
| SELL | `SELL,code,volume,price\n` | 卖出下单 |
| SUBSCRIBE | `SUBSCRIBE,code1,code2,...\n` | 订阅实时行情 |

**响应格式**：

| 响应 | 格式 | 说明 |
|------|------|------|
| PONG | `PONG\n` | 心跳响应 |
| TICK_RESULT | `TICK_RESULT,code | 最新/收盘价:x | 昨收:y\n` | tick查询结果 |
| OK | `OK\n` | 下单成功 |
| SUBSCRIBE_OK | `SUBSCRIBE_OK\n` | 订阅成功 |
| TICK | `TICK,code,price,volume,timetag\n` | 实时行情推送 |

### 客户端实现


```python
class QmtLongConnectionClient:
    def __init__(self, host='127.0.0.1', port=8888)
    def connect(self) -> bool
    def single_shot_query(self, cmd: str, timeout: float = 5.0) -> str
    def start_long_connection_loops(self)
    def send_msg(self, msg)
    def stop(self)
```

**使用流程**：

```python
# 1. 初始化并连接
client = QmtLongConnectionClient()
if not client.connect():
    return

# 2. 短链接探测（可选）
resp = client.single_shot_query("QUERY_TICK,160719.SZ\n", timeout=10.0)

# 3. 发送订单
resp_order = client.single_shot_query("BUY,160719.SZ,100,1.0\n", timeout=10.0)

# 4. 转长连接模式
client.start_long_connection_loops()
client.send_msg("SUBSCRIBE,160719.SZ,162411.SZ\n")

# 5. 保持连接并接收数据
try:
    while client.running:
        time.sleep(1)
except KeyboardInterrupt:
    pass
finally:
    client.stop()
```

```python
import sys
sys.path.insert(0, r'D:\new_tdx_test\PYPlugins\user')
from tqcenter import tq, tqconst

# 1. 初始化
tq.initialize(__file__)

# 2. 获取账户
account_id = tq.stock_account()
if account_id <= 0:
    print("获取账户句柄失败！")
    exit()

print(f"账户句柄: {account_id}")

# 3. 查询持仓
positions = tq.query_stock_positions(account_id)
for pos in positions:
    print(f"代码: {pos['Code']}, 成本: {pos['Cbj']}, 持仓: {pos['TotalVol']}")

result = tq.order_stock(
    account_id=account_id,
    stock_code="162411.SZ",
    order_type=tqconst.STOCK_BUY,
    order_volume=100,
    price_type=tqconst.PRICE_MY,
    price=0.92
)

if result.get('Value') in [1, 2] or result.get('ErrorId') == 0:
    print(f"下单成功！委托编号: {result.get('Wtbh')}")
else:
    print(f"下单失败: {result.get('Msg')}")

# 5. 查询委托
orders = tq.query_stock_orders(account_id)
for order in orders:
    print(f"委托: {order['Code']}, 状态: {order['Status']}")

# 6. 撤单（如需要）
if orders:
    order = orders[0]
    tq.cancel_order_stock(account_id, order['Code'], order['Wtbh'])

# 7. 关闭连接
tq.close()
```

## 常见问题

### 银河QMT相关

**Q: 提示连接失败，无法连接到127.0.0.1:8888？**

A: 请检查：
1. 银河QMT客户端是否已启动
2. Socket服务端策略是否已运行
3. 策略日志是否显示"QMT Socket Server Started"
4. 防火墙是否阻止了8888端口

**Q: 下单后没有反应？**

A: 请检查：
1. 账户ID是否正确
2. QMT是否已登录交易账号
3. 策略日志是否有相关错误信息

**Q: 实时行情没有推送？**

A: 请检查：
1. 是否发送了SUBSCRIBE指令
2. QMT中是否有该股票的行情数据
3. 可以尝试在QMT中切换到该股票的分时图

### 国金MiniQMT相关

**Q: 提示导入xtquant失败？**

A: 请检查：
1. QMT_INSTALL_PATH是否正确
2. 路径下是否有xtquant相关文件
3. 可以尝试手动添加路径：
   ```python
   sys.path.append(QMT_INSTALL_PATH)
   sys.path.append(os.path.join(QMT_INSTALL_PATH, "bin.x64"))
   sys.path.append(os.path.join(QMT_INSTALL_PATH, "bin.x64", "Lib", "site-packages"))
   ```

**Q: 提示userdata_mini路径不存在？**

A: 请检查：
1. QMT_INSTALL_PATH是否正确
2. MiniQMT是否已完整安装
3. 可以手动检查该路径是否存在

**Q: 连接失败，错误码非0？**

A: 请检查：
1. 国金MiniQMT客户端是否已启动并登录
2. account_id是否正确
3. 可以尝试重启MiniQMT和脚本

**Q: 下单失败？**

A: 请检查：
1. 资金是否充足
2. 股票代码格式是否正确（如160719.SZ）
3. 价格是否在涨跌停范围内
4. 是否是交易时间

**Q: 实时行情回调没有触发？**

A: 请检查：
1. 是否成功订阅
2. 股票代码格式是否正确
3. 可以尝试在MiniQMT中查看该股票是否有行情

### 通达信相关

**Q: 提示导入 tqcenter 失败？**

A: 请检查：
1. `tqcenter.py` 文件是否存在于指定路径
2. 路径 `D:\new_tdx_test\PYPlugins\user` 是否正确
3. 可以手动验证路径是否存在：
   ```python
   import os
   print(os.path.exists(r'D:\new_tdx_test\PYPlugins\user'))
   ```

**Q: 获取账户句柄失败（返回0或负数）？**

A: 请检查：
1. 通达信客户端是否已启动
2. 是否已登录交易账号
3. 账号是否有A股交易权限
4. 可以尝试手动指定账号：
   ```python
   account_id = tq.stock_account(account="您的证券账号", account_type="STOCK")
   ```

**Q: 下单失败？**

A: 请检查：
1. 资金是否充足
2. 股票代码格式是否正确（如 `162411.SZ`，带交易所后缀）
3. 价格是否在涨跌停范围内
4. 数量是否是100的整数倍
5. 是否在交易时间内
6. 查看返回的 `Msg` 字段获取详细错误信息

**Q: 如何确认下单是否成功？**

A: 检查返回值：
```python
result = tq.order_stock(...)
# 成功情况：
# - result.get('Value') in [1, 2]
# - result.get('ErrorId') == 0
# 返回委托编号：
wtbh = result.get('Wtbh')
```

**Q: 通达信下单和QMT下单可以同时使用吗？**

A: 可以，但需要注意：
1. 同一账号不能同时在两个系统登录
2. 两个客户端需要使用不同的账号
3. 建议根据券商支持情况选择主通道

### 通用问题

**Q: 应该选择哪种方案？**

A: 根据您的券商选择：
- 银河证券 → 银河QMT Socket方案
- 国金证券 → 国金MiniQMT xtquant方案
- 支持通达信的券商 → 通达信 tqcenter 方案

如果有选择余地，推荐方案优先级：
1. **国金MiniQMT**：性能最好（原生API），实现最简单
2. **通达信**：适合作为备用通道，支持极速行情
3. **银河QMT Socket**：泛用性最强，但需要维护服务端

**Q: 三种方案可以同时使用吗？**

A: 可以，但需要注意：
- 同一账号不能同时在两个交易客户端登录
- 建议选择一个主通道，其他作为备用
- 需要切换时，先关闭一个再打开另一个
